Python Faker,
Natively in the Browser

Synthora runs the actual Python Faker library client-side via Pyodide WebAssembly — no backend, no proxy, no serialization round-trips. Build-time catalog introspection delivers truthful coverage metadata at zero runtime cost.

Architecture

How Data Flows

Synthora is split into two phases: a build-time Python pipeline that produces a static catalog, and a browser runtime that loads that catalog and executes Faker methods in a dedicated Web Worker.

Design Decisions

Key Architectural Choices

Whitelist Security Model
Only methods in the pre-approved whitelist array (stored in catalog.json) can be called at runtime. The Pyodide worker validates every method name against the whitelist before execution. This prevents arbitrary code execution and ensures safe exposure of the Faker API surface.
Web Worker Isolation
Pyodide and the entire Faker runtime run inside a dedicated Web Worker. The main UI thread never blocks, even during large schema generation (1,000–5,000 rows). Progress callbacks stream back via postMessage to drive the progress bar.
Build-Time Introspection
Provider metadata (method signatures, docstrings, parameter types) is extracted at build time by a Python script that imports Faker and walks the class hierarchy. This guarantees the UI catalog is always truthful — no reflection at runtime, no stale documentation, no mismatch between displayed methods and executable ones.
Zero Backend Architecture
Served from GitHub Pages (static CDN). The Pyodide runtime + Faker wheels are fetched once and cached by the service worker. After first load, the app works fully offline. No user data ever leaves the browser — no analytics, no auth, no API calls.
Runtime

How a Method Call Works

User selects a method (e.g., email())
The React UI reads the ActiveFormatter object from the in-memory catalog. No network call needed — the catalog was already loaded on startup.
UI sends a message to the Pyodide Web Worker
A structured message { method, locale, count, seed, kwargs } is sent via postMessage. The worker queue handles concurrent requests.
Worker validates and executes
The worker checks the method name against the whitelist, constructs Faker(locale=locale, seed=seed), and calls getattr(faker, method)(**kwargs) in a loop.
Results streamed back in chunks
For large batches, results are posted back in progress chunks (every 100 rows). The UI progress bar updates in real time. Final results are serialized as JSON.
React renders the result
The Try Panel displays results in a table or JSON view. Schema Builder results are available for JSON/CSV download via PapaParse.
Getting Started

Run Locally

# Clone and install Python deps
git clone https://github.com/arvind3/Synthora.git
cd Synthora
pip install -r catalog_build/requirements.txt

# Build the method catalog (writes web/public/catalog.json)
python catalog_build/build_catalog.py

# Run Python tests
pytest catalog_build/tests

# Install frontend deps and fetch Pyodide runtime
cd web && npm install
npm run pyodide:fetch
npm run wheels:fetch

# Start dev server
npm run dev
# → http://localhost:5173/
