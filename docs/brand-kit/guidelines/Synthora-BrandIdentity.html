Naming Convention

Synthora

Always written as one word: "Synthora" — capitalized, no spaces, no camelCase. Never "Synth Ora", "SynthOra", or "SYNTHORA" (except in all-caps contexts like logos or section labels). The name is a proper noun and should always be capitalized.

Icon Concept

The Data Grid

A 3×3 grid of rounded squares inside a rounded-rectangle container — representing structured data being synthesized. One cell is accent-colored (gold) representing the moment of generation/creation. Some cells are dimmed, showing data being "filled in."

Clear Space

Minimum Padding

Maintain clear space equal to the icon height on all sides. The logo should never be crowded by other elements. On dark backgrounds, ensure the light wordmark has sufficient contrast. The icon can be used independently at small sizes.

Why These Fonts

The Rationale

Instrument Serif — Google Font, free, with an elegant italic. Evokes trust and editorial quality. DM Sans — Geometric, modern, highly legible at small sizes. Perfect for data-dense UIs. JetBrains Mono — The gold standard for developer fonts. Signals technical credibility instantly.

Color Psychology

Why This Palette

Deep Teal (#0A6E5C) — Trust, stability, and technical precision. Teal sits between blue (trust) and green (growth). It's distinctive from the sea of blue SaaS tools and connects to the "Synth" in Synthora. Midnight (#1A1A2E) — Sophistication and developer familiarity (dark mode). Gold (#F0A500) — Synthesis, creation, and warmth. It's the "spark" — the moment data comes to life.

Accessibility

Contrast Ratios

Primary teal on white: 5.2:1 (WCAG AA pass). White on midnight: 15.8:1 (AAA pass). All text combinations meet WCAG AA minimum. Gold is used only for decorative/non-text elements or large text.

05

Brand Style Guide

Consistent rules for spacing, hierarchy, and visual tone across all touchpoints.

Spacing System

Use an 8px base grid. All spacing should be multiples of 8: 8 16 24 32 48 64 80. Generous whitespace is a feature, not a waste.

Typography Hierarchy

Maximum 3 levels of heading per page. Display serif for H1 only. DM Sans for H2–H6. Minimum body font size: 14px. Line height for body: 1.6–1.7.

Border Radius

Consistent rounding system: Buttons 8px, Cards 12px, Modals 16px, Badges 6px. Never use fully rounded (pill) buttons except for icon buttons.

Icon Style

Use outline icons at 1.5px stroke weight. Lucide Icons is the recommended library. Icons should be 20px in UI and 24px in navigation. Always monochrome — never multicolor icons.

Shadows & Elevation

Three levels: Subtle 0 1px 3px rgba(0,0,0,0.06) for cards. Medium 0 4px 16px rgba(0,0,0,0.08) for dropdowns. Heavy 0 8px 32px rgba(0,0,0,0.12) for modals.

UI Tone

Clean, calm, and professional. Minimal decoration. Let content and data breathe. No gratuitous animations. Transitions should be 150ms ease for micro-interactions and 300ms ease for page transitions.

Design Principles

Dark background (Midnight #1A1A2E). Serif headline for editorial warmth. Subtle radial gradient orb behind text. Two CTAs: primary "Start Generating" + ghost "View Templates". No hero image — the product IS the hero. Link directly to the app.
